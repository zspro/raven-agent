# 01 — TUI 表格 / 富文本兼容性优化

## 问题

当前 TUI(`crates/tui/`)渲染 Markdown 富文本时兼容性不足,典型表现:

- Markdown 表格(`| a | b |`)不能正确对齐,边框错位
- 宽字符(中文、emoji)按 1 列宽计算,导致列错位
- 代码块、列表的边框/缩进渲染不一致

## 根因方向

- 终端里中文/全角字符占 2 个单元格宽,emoji 甚至更宽,需要按"显示宽度"而非"字符数"计算列宽
- 建议引入 `unicode-width` 计算字符显示宽度
